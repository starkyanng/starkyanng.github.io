准备工作

1.在github上创建一个名为username.github.io的仓库

2.将该仓库克隆到本地
$ git clone https://github.com/username/username.github.io

3.在该项目中添加一个新文件

4.提交推送到远程仓库

5.完成，在浏览器中访问[https://username.github.io]

使用jekyll生成静态网页

Github pages高度集成了Jekyll，这是一种广泛使用的静态网页生成器，也就是说，Github pages默认使用Jekyll来生成静态网页。

使用jekyll主题选择器

  1. 进入仓库的设置页面。
  2. 在“Github pages”块中，点击choose theme。
  3. 选择一个主题。
  4. 进入页面预览效果。

从jekyll迁移到hexo

比较了三个静态博客生成工具Jekyll,Octopress和Hexo,其中Octopress是基于Jekyll的，两者都是基于Ruby的，而Hexo是基于Node.js的，综合考虑，决定改用Hexo。

安装

1.环境需求

  • Node.js
  • Git

2.安装Hexo
$ npm install -g hexo-cli

3.安装成功，检测hexo版本
$ hexo version

常用命令

创建一个新post

$ hexo new [layout] <title>

创建一个新的草稿

$ hexo new draft <layout> titlename

发布一个草稿

$ hexo publish titlename

发布到github page

1.安装hexo-deploy-git
$ npm install hexo-deploy-git --save

2.修改_config.yml配置文件

3.生成静态页面
$ hexo generate

4.在本地生成站点，预览效果
$ hexo server

5.在本地预览包括草稿draft在内的效果
$ hexo server --draft

6.部署
$ hexo deploy

修改主题

1.安装主题
$ git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

2.配置主题

图片的显示问题

markdown插入图片的语法是![name](path/img.jpg)。问题在于图片的路径问题，使用了本地的相对路径后，到服务器上就显示不出来了，因为文件渲染后图片的路径变了。

解决办法是：
1.将_config.yml配置文件里的post_asset_folder属性设为true.
2.安装插件:npm install https://github.com/CodeFalling/hexo-asset-image --save
3.这样下次新创建一个post时，就会在_posts目录下创建一个与post同名的文件夹，post中所需要的资源文件就放在这个文件夹里。图片引用方式：![name](postname/img.jpg)
