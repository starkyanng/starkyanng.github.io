一个计数器counter，要实现它的计数操作counter++，其实包括以下三个步骤：

  1. 读取counter的值
  2. 将counter的值+1
  3. 更新counter的值

因此要保证计数器的线程安全，就要保证累加操作的互斥性，最简单的就是通过加锁的方式：

这样就可以保证计数器的线程安全了，但是当并发量特别大的时候，采用这种锁的方式的性能就会下降。

因此JDK中专门提供了一系列的线程安全的计数器实现，主要包括AtomicLong和LongAdder。两者都是concurrent并发包下的工具类，前者是在1.5版本中引入的，而后者是在1.8版本中引入的，拥有更高的并发性能。

然后就是compareAndSetLong:

这个方法是native的，其实底层就是硬件的CAS（compare and swap）操作，这个操作可以通过一种无锁的方式来保证更新操作的互斥性，它需要三个参数：

  • 待更新数值的内存地址
  • 该内存地址上的期望值
  • 更新后的值

如果该内存地址上的实际值等于期望值，则说明这期间没有其它线程修改过，可以更新值为新的值，返回true；否则返回false。

可以看出这个方法不能保证一次就更新成功，因此要配合的最佳实践就是如getAndAddLong方法中所示的采用while不断重试，直到成功为止。

然后其它方法和getAndAddLong()类似，都是基于CAS机制，采用无锁的方法来保证计数器在高并发量场景下的性能。
